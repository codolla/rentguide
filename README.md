Design a modern, clean, premium web application UI for a student accommodation platform called **Student Rent Guide (SRG)**.

The interface should feel like a polished SaaS product, inspired by the clean structure and simplicity of Stripe’s dashboard interface: white background, soft gray sections, clear typography, rounded cards, subtle shadows, clean spacing, minimal icons, professional layout, and excellent usability. Do not copy Stripe branding, colors, or exact screens; only use the same level of simplicity, clarity, and premium dashboard quality.

### Application Purpose

Student Rent Guide helps university students find safe and verified off-campus accommodation around USTED. The system should allow students to search for available hostels/rooms, compare prices, view room images or virtual tours, read reviews, contact landlords, and avoid accommodation fraud.

### Main Users

1. **Students** – search and compare accommodation.
2. **Landlords/Hostel Owners** – upload and manage hostel listings.
3. **Admin** – verify landlords, approve listings, manage users, and monitor reports.

### Design Style

Use a professional SaaS dashboard style:

- Clean white and light-gray background
- Soft blue or indigo primary color
- Rounded cards and buttons
- Simple line icons
- Large readable typography
- Clear sections and spacing
- Minimal but elegant interface
- Mobile responsive layout
- Trustworthy, safe, and student-friendly design

### Pages/Screens to Design

#### 1. Landing Page

Create a premium landing page with:

- Top navigation: Logo, Home, Listings, How It Works, Reviews, Login, Sign Up
- Hero section with headline: “Find Verified Student Accommodation Near Campus”
- Subtext explaining that students can search verified hostels, compare prices, view reviews, and contact landlords safely
- Search bar for location, price range, room type, and distance from campus
- CTA buttons: “Find a Room” and “List Your Hostel”
- Feature cards: Verified Listings, Student Reviews, Transparent Prices, Virtual Room Preview, Direct Landlord Contact
- Popular hostel cards
- How it works section with 3 steps: Search, Compare, Contact
- Footer with useful links

#### 2. Student Dashboard

Create a clean dashboard layout with:

- Left sidebar navigation: Dashboard, Find Rooms, Saved Listings, Messages, Reviews, Rental Guide, Profile
- Top bar with search, notification icon, and user profile
- Dashboard cards showing: Saved Hostels, New Listings, Messages, Recommended Rooms
- Recommended accommodation cards with image, hostel name, price, location, distance from campus, rating, and verification badge
- Quick filters for price, location, facilities, room type, and availability

#### 3. Accommodation Listings Page

Design a listing search page with:

- Search and filter section
- Filters: Location, Price Range, Room Type, Facilities, Distance from Campus, Rating, Availability
- Grid/list toggle
- Hostel listing cards showing:
  - Hostel image
  - Hostel name
  - Verified badge
  - Location
  - Distance from campus
  - Price per semester/year
  - Facilities icons: water, electricity, security, Wi-Fi
  - Rating and number of reviews
  - “View Details” button

#### 4. Hostel Detail Page

Create a detailed hostel page with:

- Large image gallery or virtual tour preview
- Hostel name, location, verification badge, rating
- Price details and available rooms
- Facilities section
- Description
- Map/location section
- Reviews and student feedback
- Landlord contact card
- CTA buttons: “Contact Landlord”, “Save Listing”, “Report Listing”
- Safety note reminding students to verify before payment

#### 5. Landlord Dashboard

Create a landlord dashboard with:

- Sidebar: Dashboard, My Listings, Add New Hostel, Messages, Reviews, Verification, Profile
- Cards showing Total Listings, Active Rooms, Student Inquiries, Reviews
- Table of uploaded hostels with status: Pending, Verified, Rejected
- Button to add new listing
- Simple form preview for uploading hostel details, images, price, facilities, and location

#### 6. Add Hostel Listing Page

Design a clean multi-step form:
Step 1: Hostel basic information
Step 2: Room details and pricing
Step 3: Facilities and utilities
Step 4: Upload images/video
Step 5: Submit for verification

Include form fields for hostel name, location, room type, price, water availability, electricity, security, internet, distance from campus, landlord contact, and description.

#### 7. Admin Dashboard

Create a professional admin dashboard with:

- Sidebar: Overview, Users, Landlords, Listings, Verification Requests, Reviews, Reports, Settings
- Analytics cards: Total Students, Total Landlords, Total Listings, Verified Listings, Reported Listings
- Verification request table
- Recent reports table
- Listing approval controls: Approve, Reject, View Details
- Clean charts for listing growth and user activity

#### 8. Authentication Screens

Create simple and elegant:

- Login page
- Student sign-up
- Landlord sign-up
- Forgot password
  Use clean form fields, social/email login option, and clear account type selection.

### Components Needed

Design reusable components:

- Sidebar
- Top navigation bar
- Search bar
- Filter panel
- Hostel listing card
- Rating component
- Verification badge
- Price card
- Review card
- Message card
- Admin table
- Empty state
- Alert/success/error messages

### UI Quality Requirements

The UI should look production-ready, modern, clean, and professional. It should not look like a basic school project. Make it suitable for a real startup-style student accommodation platform. Prioritize trust, clarity, safety, and ease of use. Use consistent spacing, clean hierarchy, and beautiful dashboard cards. Generate the UI in desktop and mobile responsive versions.
